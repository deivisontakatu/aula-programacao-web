# Atividade: Criação de Aplicação com Website Builders

## Objetivo

O objetivo desta atividade é que os alunos **compreendam o processo de
criação de aplicações Web utilizando Website Builders**, explorando
ferramentas que permitem desenvolver sites de forma visual e intuitiva,
sem a necessidade de programação.

A atividade tem como foco desenvolver conhecimentos sobre:

-   Website Builders
-   Criação de aplicações Web sem código
-   Uso de ferramentas baseadas em Inteligência Artificial
-   Personalização de interfaces e conteúdo
-   Análise crítica de ferramentas de desenvolvimento Web

------------------------------------------------------------------------

# Estrutura da Atividade

A atividade será dividida em duas etapas.

## 1️⃣ Aulas Teóricas -- Exploração das Ferramentas

Nas **duas primeiras aulas teóricas**, os alunos deverão:

-   Compreender o conceito de **Website Builders**
-   Explorar ferramentas de criação de sites
-   Criar uma conta em uma plataforma de Website Builder
-   Iniciar a construção de um site utilizando a ferramenta escolhida
-   Personalizar layout, cores e conteúdos do site

Durante esse período os alunos deverão **entender o processo de criação
de aplicações Web utilizando Website Builders**, observando como essas
plataformas permitem desenvolver páginas através de editores visuais e
recursos automatizados.

O professor estará disponível para orientar dúvidas sobre:

-   Funcionamento dos Website Builders
-   Escolha da ferramenta
-   Estruturação do site
-   Uso de funcionalidades interativas

------------------------------------------------------------------------

## 2️⃣ Apresentações dos Grupos

Após a etapa de desenvolvimento, **cada grupo deverá apresentar os
resultados do trabalho realizado**.

Durante a apresentação, os alunos deverão:

-   Mostrar o site desenvolvido
-   Explicar o processo de criação
-   Apresentar os desafios encontrados durante o desenvolvimento
-   Realizar uma análise crítica da ferramenta utilizada
-   Discutir os **pontos positivos e negativos da plataforma**

------------------------------------------------------------------------

# Tempo de Apresentação

Cada grupo terá aproximadamente:

**⏱ 10 a 15 minutos**

Sugestão de divisão:

-   **5 a 7 minutos** → explicação da ferramenta utilizada e do processo
    de desenvolvimento\
-   **5 a 8 minutos** → demonstração do site criado e análise do projeto

------------------------------------------------------------------------

# O que deve ser apresentado

## Ferramenta Utilizada

-   Nome do Website Builder utilizado
-   Tipo da plataforma (ex.: com IA, drag-and-drop, etc.)
-   Motivo da escolha da ferramenta

------------------------------------------------------------------------

## Processo de Desenvolvimento

Explicar:

-   Como o site foi criado
-   Como foi realizada a personalização
-   Estrutura das páginas desenvolvidas

------------------------------------------------------------------------

## Demonstração do Site

Mostrar na prática:

-   O site funcionando
-   Navegação entre páginas
-   Elementos adicionados no projeto

Exemplos de elementos que podem ser apresentados:

-   Página institucional
-   Formulários
-   Elementos interativos (quiz, simulador, calculadora ou jogo simples)
-   Layout responsivo

------------------------------------------------------------------------

## Análise da Ferramenta

Os alunos deverão apresentar uma análise sobre:

### Vantagens

Exemplos:

-   Facilidade de uso
-   Rapidez na criação
-   Recursos de inteligência artificial
-   Templates prontos

### Desvantagens

Exemplos:

-   Limitações de personalização
-   Dependência da plataforma
-   Custos de planos pagos
-   Limitações técnicas

# Observações

-   Testem o site **antes da apresentação**.
-   Garantam que o projeto esteja **funcionando corretamente**.
-   Organizem a apresentação para abordar **processo, resultado e
    análise crítica**.
-   Caso o site não funcione durante a apresentação, a avaliação poderá
    ser comprometida.
