# ⚡ Sistemas Web com Chamadas Assíncronas

Chamadas assíncronas permitem que partes da aplicação sejam atualizadas sem recarregar toda a página.

---

## 🔄 Conceito

Execução de operações em segundo plano, sem bloquear o fluxo principal.

---

## 🌐 Tecnologias

### 📡 AJAX

* Comunicação assíncrona com o servidor
* Atualiza partes da página

### 📦 Fetch API

* Alternativa moderna ao AJAX

```javascript
fetch('/api/usuarios')
  .then(response => response.json())
  .then(data => console.log(data));
```

---

### 🔌 APIs REST

* Comunicação entre frontend e backend
* Retornam dados (JSON)

---

## ⚙️ Integração com Banco de Dados

Fluxo típico:

1. Usuário interage com a interface
2. Requisição assíncrona é enviada
3. Servidor consulta o banco
4. Retorna dados (JSON)
5. Interface é atualizada dinamicamente

---

## 🎯 Vantagens

* Melhor experiência do usuário (UX)
* Redução de recarregamentos
* Aplicações mais rápidas e dinâmicas

---

## ⚠️ Desafios

* Controle de concorrência
* Tratamento de erros
* Segurança (autenticação, CORS)

---

## 🔗 Exemplos de Uso

* Atualização de feed em tempo real
* Autocomplete de busca
* Carrinho de compras dinâmico

---

## ✅ Resumo

* Assíncrono = não bloqueante
* Base de aplicações modernas (SPA)
* Integra frontend, backend e banco de dados
