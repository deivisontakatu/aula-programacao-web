# 🏗️ 06.2 - Arquitetura de Sistemas Web Escaláveis

A arquitetura de sistemas web escaláveis é responsável por garantir que aplicações suportem crescimento de usuários, volume de dados e requisições sem perda de desempenho.

---

## 🎯 Objetivo

Projetar sistemas capazes de:
- Suportar alta carga de acessos
- Manter desempenho estável
- Crescer de forma sustentável
- Garantir disponibilidade

---

## 📈 O que é Escalabilidade?

Escalabilidade é a capacidade de um sistema aumentar sua capacidade conforme a demanda.

---

## 🔹 Tipos de Escalabilidade

### 🔼 Escalabilidade Vertical
- Aumenta recursos da máquina (CPU, RAM)
- Simples, porém limitada

---

### 🔀 Escalabilidade Horizontal
- Adiciona novos servidores
- Mais robusta e distribuída

---

## 🧱 Arquiteturas Comuns

### 🧩 Monolítica
- Sistema único
- Simples de desenvolver
- Difícil de escalar

---

### 🔗 Microsserviços
- Sistema dividido em serviços independentes
- Alta escalabilidade
- Maior complexidade

---

### 🌐 Arquitetura em Camadas

- Apresentação (Frontend)
- Aplicação (Backend)
- Dados (Banco)

---

## ⚖️ Balanceamento de Carga

Distribui requisições entre múltiplos servidores.

### Benefícios:
- Melhor desempenho
- Alta disponibilidade
- Redução de falhas

---

## 🔗 Proxy Reverso

Servidor intermediário que:
- Recebe requisições
- Encaminha para serviços internos

---

## 🧊 Uso de Containers

- Isolamento de serviços
- Facilidade de deploy
- Escalabilidade rápida

---

## ☁️ Computação em Nuvem

Permite:
- Escalar sob demanda
- Alta disponibilidade
- Redução de custos operacionais

---

## 🛢️ Banco de Dados Escalável

### Estratégias:
- Replicação (read replicas)
- Sharding (divisão de dados)
- Cache (Redis)

---

## ⚡ Cache

Armazena dados frequentemente acessados.

### Benefícios:
- Redução de carga no banco
- Respostas mais rápidas

---

## 📡 Comunicação entre Serviços

- APIs REST
- Mensageria (RabbitMQ, Kafka)

---

## 🔄 Processamento Assíncrono

- Filas de mensagens
- Workers

---

## 🔐 Segurança em Escala

- Autenticação distribuída
- Tokens (JWT)
- Gateways de API

---

## 📊 Monitoramento

- Logs centralizados
- Métricas (CPU, memória)
- Alertas

---

## 🚀 Alta Disponibilidade

- Redundância de servidores
- Failover automático

---

## ⚠️ Desafios

- Complexidade
- Consistência de dados
- Monitoramento

---

## 🧠 Exemplo de Arquitetura

- Frontend (CDN)
- API Gateway
- Microsserviços
- Banco distribuído
- Cache

---

## 🎯 Boas Práticas

- Stateless services
- Separação de responsabilidades
- Uso de cache
- Automação

---

## 🎯 Conclusão

Sistemas escaláveis são essenciais para aplicações modernas com alto volume de usuários.
