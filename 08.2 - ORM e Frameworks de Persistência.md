# 🧩 08.2 - ORM e Frameworks de Persistência

ORM (Object-Relational Mapping) permite mapear objetos de uma aplicação para tabelas do banco de dados.

---

## 🔄 Conceito de ORM

* Classes → Tabelas
* Objetos → Registros
* Atributos → Colunas

---

## 🎯 Vantagens

* Reduz código SQL manual
* Aumenta produtividade
* Facilita manutenção
* Abstrai complexidade do banco

---

## ⚠️ Desvantagens

* Pode gerar consultas ineficientes
* Menor controle sobre SQL
* Curva de aprendizado

---

## 🛠️ Frameworks Populares

### ☕ Java

* Hibernate
* JPA

### 🐍 Python

* SQLAlchemy
* Django ORM

### 🌐 JavaScript

* Sequelize
* TypeORM

---

## 🧠 Exemplo Conceitual

```java
Usuario user = new Usuario("João");
entityManager.persist(user);
```

→ O ORM converte isso em SQL automaticamente.

---

## 🧱 Arquitetura com ORM

* Model (entidade)
* Repository/DAO
* Serviço (lógica de negócio)

---

## ✅ Resumo

* ORM conecta mundo orientado a objetos com banco relacional
* Aumenta produtividade, mas exige cuidado com performance
