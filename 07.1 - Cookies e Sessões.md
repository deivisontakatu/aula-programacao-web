# 🍪 Cookies e Sessões

Como o HTTP é stateless, são necessários mecanismos para manter o estado do usuário. Os principais são **cookies** e **sessões**.

---

## 🍪 Cookies

Cookies são pequenos arquivos armazenados no navegador do usuário.

### 📌 Características:

* Armazenam dados simples (ex: preferências, identificadores)
* São enviados automaticamente ao servidor em cada requisição
* Possuem tempo de expiração

### 📊 Tipos:

* **Sessão:** expiram ao fechar o navegador
* **Persistentes:** permanecem por tempo definido

### ⚠️ Boas práticas:

* Não armazenar dados sensíveis
* Utilizar `HttpOnly` e `Secure`

---

## 🔐 Sessões

Sessões permitem manter informações do usuário no servidor.

### ⚙️ Funcionamento:

1. Usuário acessa o sistema
2. Servidor cria uma sessão com ID único
3. ID é enviado ao cliente (geralmente via cookie)
4. A cada requisição, o servidor identifica o usuário pela sessão

---

## 🧠 Escopos de Memória

### 🟢 Aplicação (Application Scope)

* Compartilhado entre todos os usuários
* Dura enquanto a aplicação estiver ativa

### 🔵 Sessão (Session Scope)

* Específico por usuário
* Dura durante a sessão

### 🟡 Request (Request Scope)

* Existe apenas durante uma requisição

---

## 🔗 Relação entre Cookies e Sessões

* O **cookie armazena o ID da sessão**
* A **sessão armazena os dados no servidor**
* Juntos, permitem manter o estado do usuário

---

## 🛢️ Integração com Banco de Dados

Sessões podem ser integradas com banco de dados para:

* Persistir login
* Armazenar dados do usuário
* Controlar autenticação

---

## ✅ Resumo

* Cookies → armazenados no cliente
* Sessões → armazenadas no servidor
* HTTP → stateless
* Sessões + Cookies → mantêm estado da aplicação
